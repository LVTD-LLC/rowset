Workflow

The handoff is small.

Sign in, copy the setup prompt, give a trusted agent a bearer key, and let it use Rowset over MCP or REST.

MCP access Dataset API

01

Connect

The agent stores ROWSET_API_KEY privately and verifies access.

02

Create

It creates datasets with headers, an index column, and useful context.

03

Operate

It reads, updates, exports, and shares rows inside your account boundary.

Why Rowset

Agents need state, not another fixed app.

Shape changes by job

The agent can create a better dataset when the workflow changes.

Machine-readable truth

Keys, headers, types, metadata, and URLs stay explicit.

Human control surface

The dashboard is for setup, review, exports, and previews.

No public scraping path

Private MCP and REST access stay separate from public previews.
