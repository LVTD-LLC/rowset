# Rowset Brand

## Product

Rowset gives trusted AI agents a stable backend for user-owned structured datasets.
Users can sign in, copy an agent setup prompt, connect over hosted MCP or REST with
a bearer API key, and let agents create, inspect, update, export, and share rows.

## Primary Persona

Technical founders, operators, analysts, engineers, and agent-heavy small teams who
want agents to work with structured business data without building a custom backend
or forcing humans through upload wizards.

## Positioning

- Agent-native dataset backend.
- Private by default, with explicit read-only public previews for human review.
- MCP first, REST second, browser automation last.
- Structured rows, stable index columns, semantic schema metadata, exports, and
  project grouping.
- A practical alternative when Google Sheets is too fragile and a warehouse or
  internal platform is too heavy.

## Anti-Positioning

- Not a spreadsheet replacement for humans.
- Not a BI dashboard or warehouse.
- Not a source-owned sync platform for Google Sheets, CRMs, or databases.
- Not a public data host by default.
- Not a way to bypass ownership, authentication, or review boundaries.

## Differentiators

- Gives agents a durable MCP and REST surface for owned datasets.
- Lets agents create the dataset instead of asking humans to upload files.
- Separates private programmatic access from public preview sharing.
- Supports CSV, JSONL, XLSX, SQLite, and Parquet export paths.
- Includes setup prompts and capability discovery so agents can self-configure.

- Data warehouse plus semantic layer products

## Voice

Write like a technical operator explaining a useful data workflow to another
operator:

- Concrete, calm, and low-hype.
- Specific about data shape, auth, and ownership.
- Honest about what Rowset does not own.
- Short paragraphs, practical examples, and clear conversion paths.
- Use nouns like dataset, row, index column, MCP tool, REST endpoint, API key,
  public preview, export, and schema metadata.

## Forbidden Words and Moves

- Do not call Rowset magic, revolutionary, effortless, game-changing, or
  AI-powered as filler.
- Do not imply public previews are private authentication.
- Do not imply Rowset syncs Google Sheets or CRMs directly as a current product
  capability.
- Do not invent customer stories, benchmarks, revenue, security audits, or
  usage metrics.
- Do not publish private dataset contents, API keys, or unmasked secrets.
